## 第1章 机器学习概览（完整总结）

### 1. 什么是机器学习
- **学术定义**（Arthur Samuel, 1959）：让计算机在不依赖明确编程的情况下具备学习能力。
- **工程化定义**（Tom Mitchell, 1997）：一个计算机程序在任务T上，通过经验E提升性能P，即可称为学习。
- **核心区别**：下载维基百科只是存储数据，不提升任务表现，因此不属于机器学习。

### 2. 为什么要使用机器学习
- **简化复杂规则**：传统编程需要手动编写大量规则（如垃圾邮件过滤），难以维护和扩展；机器学习可自动从数据中学习模式。
- **解决传统方法无法处理的问题**：如语音识别、图像理解等，无法通过硬编码规则实现。
- **适应环境变化**：当垃圾邮件发送者改变策略时，传统过滤器需要手动更新规则，而机器学习系统可自动适应新数据。
- **辅助人类学习**：通过数据挖掘发现人类难以察觉的模式和关联，帮助理解问题本质。

### 3. 机器学习系统的种类
根据三个维度进行分类，各维度可自由组合：

#### （1）监督式 / 无监督式 / 半监督式 / 强化学习
- **监督式学习**
  - 训练数据包含**标签**（正确答案）。
  - 典型任务：
    - **分类**：预测类别（如垃圾邮件分类）。
    - **回归**：预测连续值（如汽车价格预测）。
  - 常见算法：K-近邻、线性回归、逻辑回归、SVM、决策树与随机森林、神经网络。

- **无监督式学习**
  - 训练数据**无标签**，系统自主学习数据结构。
  - 典型任务：
    - **聚类**：发现数据分组（如博客访客分组）。
    - **可视化与降维**：如t-SNE可视化、PCA降维，简化数据同时保留结构。
    - **关联规则学习**：发现特征间的关联（如购物篮分析）。
    - **异常检测**：识别离群点（如信用卡欺诈检测）。
  - 常见算法：k-Means、HCA、EM、PCA、t-SNE、Apriori等。

- **半监督式学习**
  - 结合少量标签数据和大量无标签数据（如深度信念网络）。
  - 例子：Google相册自动聚类人物，再由用户标记。

- **强化学习**
  - 智能体通过与环境交互，学习最优策略以最大化奖励。
  - 例子：AlphaGo、机器人行走控制。

#### （2）批量学习与在线学习
- **批量学习**
  - 一次性使用全部训练数据训练模型，无法动态更新。
  - 适合数据量不大、变化不频繁的场景。

- **在线学习**
  - 逐次或分批接收数据，实时更新模型。
  - 适合数据流、资源受限或需要快速响应的场景。
  - 关键参数：**学习率**（控制适应新数据的速度）。
  - 挑战：不良数据会导致性能下降，需要监控和异常检测。

#### （3）基于实例与基于模型的学习
- **基于实例学习**
  - 通过记忆训练数据，对新数据进行相似度匹配（如K-近邻）。
  - 优点：简单直观；缺点：泛化能力有限，计算量大。

- **基于模型学习**
  - 从数据中学习通用模式，构建预测模型（如线性回归）。
  - 典型流程：
    1. 选择模型（如线性模型：`生活满意度 = θ₀ + θ₁ × 人均GDP`）。
    2. 定义成本函数（如均方误差）。
    3. 训练算法找到最优参数（最小化成本）。
    4. 应用模型进行预测。
  - 例子：用Scikit-Learn训练线性回归模型预测塞浦路斯的生活满意度。

### 4. 机器学习的主要挑战

#### （1）训练数据的数量不足
- 大多数算法需要大量数据，复杂任务（如图像识别）甚至需要上千万个示例。
- 研究表明：对复杂问题，**数据比算法更重要**，但获取高质量数据往往成本高昂。

#### （2）训练数据不具代表性
- 数据分布与实际应用场景差异大，导致泛化能力差。
- 常见问题：
  - **采样噪声**：样本集太小，导致非代表性数据被选中。
  - **采样偏差**：采样方式不当，如1936年《Literary Digest》民调错误预测大选结果。

#### （3）质量差的数据
- 包含错误、异常值、噪声或缺失值，影响模型学习。
- 处理方法：
  - 丢弃或修复异常值。
  - 填充缺失值（如中位数）或忽略缺失实例。

#### （4）无关特征
- 过多无用特征会干扰模型，需要**特征工程**筛选：
  - 特征选择：从现有特征中选择最有用的。
  - 特征提取：整合现有特征（如降维）。
  - 收集新数据创造新特征。

#### （5）过度拟合
- 模型在训练集上表现极好，但在新数据上表现差，通常是因为模型过于复杂。
- 解决方法：
  - 简化模型（如选择线性模型而非高阶多项式）。
  - 收集更多训练数据。
  - 减少噪声（修复错误、消除异常值）。
  - **正则化**：通过约束模型参数，降低复杂度（如线性模型中限制斜率θ₁）。

#### （6）拟合不足
- 模型过于简单，无法捕捉数据中的复杂模式。
- 解决方法：
  - 选择更强大的模型（增加参数）。
  - 提供更好的特征集（特征工程）。
  - 减少模型约束（降低正则化强度）。

### 5. 测试与验证
- **核心目标**：评估模型的泛化能力，避免过度拟合。
- **数据分割**：
  - **训练集**（80%）：用于训练模型。
  - **测试集**（20%）：用于评估泛化误差。
- **避免数据泄露**：
  - 不要用测试集调整超参数，否则会导致模型“记住”测试集。
  - 解决方案：引入**验证集**，或使用**交叉验证**（将训练集分成多个子集，轮流作为验证集）。
- **No Free Lunch定理**：没有一种算法在所有问题上都最优，必须根据具体问题选择模型并做出假设。

### 6. 典型机器学习项目流程
1. 学习数据（收集、清洗、探索）。
2. 选择模型。
3. 使用训练数据进行训练（最小化成本函数）。
4. 应用模型对新示例进行预测（推断）。
